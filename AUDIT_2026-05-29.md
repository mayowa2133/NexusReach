## ЁЯФ┤ CRITICAL

### C1. SearXNG results systematically penalized in people scoring тЬЕ
**Files:** `backend/app/services/people_service.py` lines ~115-143
`CURRENT_TRUSTED_SOURCES` includes `brave_hiring_team` / `serper_hiring_team` but NOT `searxng_hiring_team`. `SOURCE_PRIORITY` lacks all `searxng_*` labels. SearXNG is the default primary provider, so every person it discovers gets worst-rank priority 5 and is treated as untrusted. The system penalizes its own primary pipeline.

### C2. Bucket name typo breaks peer title backfill тЬЕ
**File:** `backend/app/services/people_service.py` line ~1502
`bucket != "peer"` should be `bucket != "peers"`. Codebase uses plural buckets everywhere, so the condition is always True and backfilled titles unconditionally overwrite existing peer titles.

### C3. `search_jobs` returns only NEW jobs, not refreshed/matched ones тЬЕ
**File:** `backend/app/services/job_service.py` lines ~682-692
Existing jobs are updated in place by `_refresh_existing_job` but never appended to `stored_jobs`. `POST /jobs/search` returns `[]` when all results dedup-match existing rows. Search feels broken on repeat searches.

### C4. Hardcoded Dice API key committed to source тПня╕П (code done; needs env var + key rotation тАФ see AUDIT_FIX_NOTES.md)
**File:** `backend/app/clients/remote_jobs_client.py` line ~26
`DICE_HEADERS` contains an inline `x-api-key`. Move to env config.

### C5. Adzuna currency mapping wrong for non-US/GB countries тЬЕ
**File:** `backend/app/clients/adzuna_client.py` line ~64
`"USD" if country == "us" else "GBP" if country == "gb" else "USD"` mislabels CA/AU/EU/IN salaries as USD.
